## 4. VoiceCore와 인프라 공유 전략

VoiceCore (Fish Speech S2)는 이미 RTX 4090 + RunPod 아키텍처로 설계됨. 동일 GPU에서 시간 분할(Time-sharing) 운영 가능.

### 4.1 GPU 시간 예산 (Scale2 기준, 월 720hr)

| 작업 | GPU-hr/월 | 점유율 |
|------|----------|--------|
| VoiceCore TTS | ~45hr (26,900분 ÷ 600분/hr) | 6% |
| ImageCore 이미지 | ~249hr | 35% |
| VideoCore 영상 | ~178hr (생성+업스케일+재시도) | 25% |
| **합계** | **~472hr** | **66%** |
| 여유 | ~248hr | 34% |

**결론: RTX 4090 1대로 TTS + 이미지 + 영상 모두 처리 가능.**

단, Seed~Scale1까지는 1대로 충분하지만, Scale2에서 피크 시간대에 병목이 발생할 수 있으므로 GPU 2대 병렬 운영을 권장.

### 4.2 운영 아키텍처

```
RunPod RTX 4090 (GPU-1: Primary)
├── Fish Speech S2 (VoiceCore) — Docker, port 8080
├── Flux.1 Dev (ImageCore) — ComfyUI API, port 8188
└── HunyuanVideo 1.5 (VideoCore) — REST API, port 8190

RunPod RTX 4090 (GPU-2: Overflow, Scale1+ 투입)
├── HunyuanVideo 1.5 (VideoCore 영상 전용)
└── Real-ESRGAN (업스케일 전용)

N8N WF-2 스크립트 생성 완료
    ↓
N8N WF-3 TTS (VoiceCore API → 음성 파일)
    ↓
N8N WF-IMAGE (ImageCore API → 스틸 이미지)
    ↓
N8N WF-VIDEO (VideoCore API → 영상 클립, 20% 씬만)
    ↓
N8N WF-RENDER (Shotstack → 최종 조립)
    ↓
N8N WF-UPLOAD (업로드)
```
