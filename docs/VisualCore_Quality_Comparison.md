### 2.3 핵심 격차 분석

**Seedance (12.88) vs HunyuanVideo (11.3) = 약 12% 차이**

이 12% 차이가 270ch 유튜브에서 의미하는 바:

| 관점 | Seedance 우위 | HunyuanVideo 수용 가능성 | 판단 |
|------|-------------|----------------------|------|
| 모션 자연스러움 | ✅ 확실히 우수 | 보조 씬에서 허용 가능 | ⚠️ 핵심씬은 Seedance 유지 |
| 물리 시뮬레이션 | ✅ 더 정교 | 단순 카메라 워크는 OK | ⚠️ 복잡한 물리는 Seedance |
| 멀티샷 일관성 | ✅ 네이티브 지원 | 미지원 | ⚠️ 멀티샷은 Seedance |
| 인물 표현 | ✅ 전반적 우수 | **인물 충실도에서 HunyuanVideo가 강점** | ✅ 인물 클로즈업 가능 |

**이미지와 영상의 격차가 다른 이유:**
- 이미지: 정적 화면, 3~6초 표시 → 미세한 품질 차이 인지 불가
- 영상: 동적 화면, 모션/물리/일관성이 모두 중요 → 차이가 더 크게 드러남

### 2.4 HunyuanVideo 1.5의 기술적 장점

기존 오픈소스 영상 모델 대비 HunyuanVideo 1.5가 선택된 이유:

| 지표 | HunyuanVideo 1.5 | Wan 2.2 | Open-Sora 2.0 |
|------|-----------------|---------|---------------|
| 파라미터 | **8.3B (가벼움)** | 27B (14B active) | 11B |
| VRAM | **14GB (offload)** | 24GB+ | 24GB |
| 속도 (RTX 4090) | **~19초/5초클립** | ~38초/5초클립 | ~60초/5초클립 |
| 품질 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐☆ (약간 우세) | ⭐⭐⭐ |
| 인물 표현 | **강점** | 보통 | 보통 |
| Step Distillation | **✅ (75% 가속)** | ❌ | ❌ |
| Consumer GPU | **✅ RTX 4090 OK** | ⚠️ 빡빡 | ⚠️ 빡빡 |

Wan 2.2가 시네마틱 품질에서 약간 우세하지만, **속도 2배 느리고 VRAM 70% 더 필요**. 
270ch 대량 생산 환경에서는 HunyuanVideo의 효율성이 더 중요.

## 6. 최종 권장

### 6.1 즉시 실행

- **이미지**: Flux Klein 4B로 **100% 대체** (자신 있음)
- **영상**: HunyuanVideo 1.5로 **80% 대체**, 핵심 씬 20%만 Seedance API 유지
- **썸네일**: Flux Klein + 텍스트 LoRA로 대체 (PoC 결과에 따라 Ideogram 유지 가능)

### 6.2 6개월 후 재평가

- Flux Klein → Klein 9B 또는 Qwen Image 업그레이드 검토
- HunyuanVideo → 후속 모델로 영상 품질 보완
- Seedance API 비중 20% → 10% → 0% 점진 축소

### 6.3 리스크 헷지

- QC 파이프라인 (CLIP Score + Aesthetic)으로 품질 하한선 자동 보장
- 3회 재생성 후에도 QC 실패 시 API fallback → 품질 저하가 시청자에게 노출되지 않음
- LoRA 파인튜닝으로 니치별 품질 보완 (특히 T1 Space, T7 Crime 등 시각적 특화 니치)
